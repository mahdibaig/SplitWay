# Splitway

Native iOS app for tracking and splitting household expenses. Built with SwiftUI, Core Data, and CloudKit.

See [Design/project/uploads/splitway_package/APP_SUMMARY.md](Design/project/uploads/splitway_package/APP_SUMMARY.md) for product vision, and [Design/project/uploads/splitway_package/PHASE_1_FOUNDATION.md](Design/project/uploads/splitway_package/PHASE_1_FOUNDATION.md) for the current build phase.

## Layout

```
.
├── project.yml              xcodegen config, single source of truth for the Xcode project
├── Splitway/                Swift sources (compiled into the app target)
│   ├── App/                 entry point + launch routing
│   ├── DesignSystem/        colors, typography, spacing tokens
│   ├── Models/CoreData/     Splitway.xcdatamodeld (User, Household, Group)
│   ├── Services/            PersistenceController + CloudKit + business logic
│   ├── Views/               Onboarding, Home, Settings
│   └── Resources/           Assets.xcassets
├── Splitway.xcodeproj/      generated by xcodegen (do not edit by hand)
└── Design/                  design handoff: mockups, spec docs, tokens.js
```

## Regenerating the Xcode project

The `.xcodeproj` is generated from `project.yml`. After adding or removing files, run:

```sh
xcodegen generate
```

You can also commit `project.yml` and ignore the generated `.xcodeproj` (already in `.gitignore`).

## Building

1. Open `Splitway.xcodeproj` in Xcode.
2. Select your team under Signing & Capabilities (Apple ID works for simulator; paid Developer Program needed for CloudKit on a real device).
3. Pick an iPhone simulator and Run.

CloudKit features will not function until the iCloud container `iCloud.com.mahdibaig.splitway` is created in the Apple Developer portal and a real iCloud-signed-in device is used.
